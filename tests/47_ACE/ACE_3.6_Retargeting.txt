# Wiki : https://wiki.4info.com/display/Software/ACE+API+Documentation#ACEAPIDocumentation-3.6Retargeting(4INFORetargeting,CatalinaRetargeting)
# API Name : Create Retargeting Segment.
# Method : POST
# URL : qa-ace-10:8080/ace/api/v2/report/create
# DESC : 4INFO and Catalina Retargeting differ only in which group has access
#        to this function and the created Retargeting Segment.
#        4INFO Retargeting Segments are accessible to users belonging to the 4INFO group,
#        and similarly Catalina Retargeting Segments are accessible to users belonging to
#        the Catalina group. From the API standpoint, the two differ only in the
#        data source ID passed in the data_source parameter.
#        This causes their target_datasource_segment records to be created under the
#        respective data sources. The created Retargeting Segments are accessible to the
#        respective groups. Other than that, the functionality is exactly the same for
#        both jobs.

# rhino.ace_report
