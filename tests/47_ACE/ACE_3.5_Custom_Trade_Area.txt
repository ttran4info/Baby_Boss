# Wiki : https://wiki.4info.com/display/Software/ACE+API+Documentation#ACEAPIDocumentation-3.5CustomTradeArea(4INFOCustomTradeArea,CatalinaCustomTradeArea)
# API Name : Create Custom Trade Area (4INFO Custom Trade Area, Catalina Custom Trade Area)
# Method : POST
# URL : qa-ace-10:8080/ace/api/v2/report/create
# DESC : 4INFO and Catalina Custom Trade Area differ only in which group has access
#        to this function and the created Custom Trade Area.
#        4INFO Custom Trade Areas are accessible to users belonging to the 4INFO group,
#        and similarly Catalina Custom Trade Areas are accessible to users belonging to
#        the Catalina group. From the API standpoint,
#        the two differ only in the data source ID passed in the dataSource parameter.
#        This causes their target_datasource_segments to be created under the respective
#        data sources. The created Custom Trade Areas are accessible to the respective
#        groups. Other than that, the functionality is exactly the same for both jobs.
#        Instead of passing a list of POIs in 'segmentEntryList' parameter as is done
#        for Trade Area; for Custom Trade Area, a .csv file with the list of POIs is
#        uploaded by the API to Amazon S3, and downloaded from S3 to the ACE server.
#        The file  is passed in 'fileName' parameter.
#        The rest of the Custom Trade Area creation is the same as Geofence.
